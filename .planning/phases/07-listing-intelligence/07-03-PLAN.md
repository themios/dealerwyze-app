Build the MLS# import path (LIST-03), listing performance metrics API (LIST-04), and CMA generation API (LIST-05). These three routes complete the server-side API layer for Phase 7. The RentCast wrapper is shared between MLS import and CMA generation.

This plan runs in parallel with plan 07-02 (different files, no file conflicts).

**MLS import saves immediately** — unlike the URL and photo import paths (which return extracted data for form preview), the MLS route creates a vehicles row directly and returns the new record id. The UI redirects the agent to the listing detail page where they can review and edit the RentCast-populated fields. This is the correct behavior for MLS# import: the MLS# + address are authoritative inputs, so no preview step is needed.

Purpose: Agents can import a listing via MLS# + address, view performance metrics, and generate comparable market analyses from RentCast — all without leaving RealtyWyze.

  <action>
    Create the RentCast API wrapper. All calls require the `X-Api-Key` header. Validate the env var exists at call time — throw a descriptive error if missing (caller converts to 503).

    **Exports:**

    `fetchPropertyByAddress(address: string): Promise<RentCastProperty | null>`
    - Calls `GET https://api.rentcast.io/v1/properties?address={encoded}&limit=1`
    - Returns the first result or null if no match
    - Type `RentCastProperty` includes: `address`, `city`, `state`, `zipCode`, `bedrooms`, `bathrooms`, `squareFootage`, `lotSize`, `yearBuilt`, `propertyType`, `mlsNumber` (optional)
    - On non-200 response, log the status and throw `new Error('RentCast lookup failed: ${status}')`

    `fetchCMA(address: string, opts?: { bedrooms?: number, bathrooms?: number, sqft?: number, propertyType?: string }): Promise<RentCastCMA>`
    - Calls `GET https://api.rentcast.io/v1/avm/value?address={encoded}&compCount=15&...opts`
    - Required: full address string in format "Street, City, State Zip" (e.g. "123 Main St, Portland, OR 97201")
    - Include opts params if provided (bedrooms, squareFootage, propertyType)
    - Type `RentCastCMA` includes: `price`, `priceRangeLow`, `priceRangeHigh`, `comparables: Array<{ address, price, squareFootage, bedrooms, bathrooms, distance, correlation }>`
    - On non-200 response: throw `new Error('RentCast CMA failed: ${status}')`

    **Helper `buildFullAddress(parts: { address_line1: string; city: string; state: string; zip: string }): string`:**
    Returns `"${address_line1}, ${city}, ${state} ${zip}"`. Validate all parts are non-empty; throw if any is missing.

    **RENTCAST_API_KEY guard:**
    ```typescript
    function getRentCastKey(): string {
      const key = process.env.RENTCAST_API_KEY
      if (!key) throw new Error('RENTCAST_API_KEY is not configured. Contact your administrator.')
      return key
    }
    ```
    Call this inside each exported function, not at module level (allows build to succeed without the key set).

    ---

    **app/api/listings/import-mls/route.ts (LIST-03):**

    This route creates a new vehicles row from MLS# + address input. The agent provides both MLS# and address (since RentCast does not accept MLS# as a primary lookup key — it must be address-based).

    **Important:** This route saves immediately — it does not return a preview payload. After success, the UI redirects the agent to `/vehicles/{id}` where they can review and edit the RentCast-populated fields. This is intentional: MLS# + address are authoritative inputs that do not require a preview step.

    Flow:
    1. `requireProfile()` → vertical guard
    2. Validate body: `{ mls_number: string, address_line1: string, city: string, state: string, zip: string }` (all required)
    3. Call `fetchPropertyByAddress(buildFullAddress(...))` from rentcast.ts
    4. If RentCast returns data: merge returned property fields with agent-provided address + MLS#
    5. If RentCast returns null: proceed with just the agent-provided fields (RentCast is best-effort)
    6. If RENTCAST_API_KEY missing: return 503 "RentCast is not configured. Enter listing details manually."
    7. Build the vehicles insert payload:
       - `user_id: profile.org_id`
       - `year: 0, make: 'RE', model: address_line1.slice(0, 100)` — required NOT NULL placeholders
       - `stock_no: \`MLS-${mls_number}-${Date.now()}\`` — unique placeholder
       - `status: 'active'`
       - `mls_number` from input
       - `address_line1, city, state, zip` from input
       - `bedrooms, bathrooms, sqft, lot_size, year_built, property_type` from RentCast response (if available)
       - `import_source: 'mls_import'`
       - `import_raw_json`: the raw RentCast response (for debugging)
    8. Insert to vehicles using `createClient()` (RLS scopes to org automatically)
    9. Return `{ id: newRow.id, ...extractedFields }` with 201

    Zod validation schema for request body:
    ```typescript
    const schema = z.object({
      mls_number: z.string().min(1).max(50),
      address_line1: z.string().min(5).max(200),
      city: z.string().min(1).max(100),
      state: z.string().length(2),
      zip: z.string().regex(/^\d{5}$/),
    })
    ```

    **app/api/listings/[id]/metrics/route.ts (LIST-04):**

    Returns performance metrics for a listing. Reads denormalized fields from vehicles (showing_count, price_change_count, price_change_log added by migration 189) plus calculated days_on_market.

    Flow:
    1. `requireProfile()` → vertical guard
    2. Fetch vehicle by id, filtered by `user_id = profile.org_id`:
       `SELECT id, status, price, created_at, showing_count, price_change_count, price_change_log, listing_type FROM vehicles WHERE id = $id AND user_id = $org_id`
    3. If not found: 404
    4. Calculate `days_on_market = Math.floor((Date.now() - new Date(vehicle.created_at).getTime()) / 86_400_000)`
    5. Return:
       ```json
       {
         "days_on_market": number,
         "showing_count": number,
         "price_change_count": number,
         "price_change_log": [{ "from": number, "to": number, "changed_at": string }] | null,
         "current_price": number,
         "status": string
       }
       ```

    ---

    **app/api/listings/[id]/cma/route.ts (LIST-05):**

    Generates or returns cached CMA. Adapts `market-check/route.ts` for RentCast. Cache TTL: 168 hours (7 days), same as dealer market check.

    Flow:
    1. `requireProfile()` → vertical guard
    2. Fetch vehicle fields needed for CMA: `address_line1, city, state, zip, bedrooms, bathrooms, sqft, property_type, market_data_json, market_checked_at`
    3. Verify `user_id = profile.org_id` (404 if not found)
    4. Check cache: if `market_checked_at` exists and age < 168h AND `market_data_json` is present → return `{ data: market_data_json, cached: true }`
    5. Validate full address available (all 4 parts non-empty) → return 422 with "Complete the property address before running a CMA" if not
    6. Call `fetchCMA(buildFullAddress(...), { bedrooms, bathrooms, sqft, propertyType: property_type })`
    7. If RENTCAST_API_KEY missing: return 503 with clear message
    8. On success: `UPDATE vehicles SET market_data_json = cmaData, market_checked_at = now() WHERE id = id`
    9. Return `{ data: cmaData, cached: false }`

    Set `export const maxDuration = 30` (RentCast is fast; 30s is generous).
