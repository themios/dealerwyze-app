  <name>Task 1: POST /api/showings (SHOW-01) + GET /api/showings?listing_id=X (SHOW-04 data layer)</name>
  <files>app/api/showings/route.ts</files>
  <action>
Create `app/api/showings/route.ts` with `export const runtime = 'nodejs'`.

**POST handler — create showing:**

1. `requireProfile()` first. Use `createClient()` (not createServiceClient — user session exists).
2. Parse JSON body. Validate with inline Zod schema:
   ```typescript
   const schema = z.object({
     listing_id:   z.string().uuid(),
     scheduled_at: z.string().datetime(),
     contact_id:   z.string().uuid().optional(),
     notes:        z.string().max(2000).optional(),
   })
   ```
3. Insert into showings: `org_id = profile.org_id`, `agent_id = profile.id`, `status = 'scheduled'`. Do NOT set showing_count — trigger handles it.
4. On successful insert, attempt GCal sync best-effort:
   - Fetch listing address: query `vehicles` table with `eq('id', body.listing_id).eq('org_id', profile.org_id)` — 404 if not found (ownership check).
   - Call `createCalendarEvent({ summary: \`Showing: ${address}\`, description: notes ?? '', startDateTimeIso: body.scheduled_at }, profile.org_id)`.
   - If `gcal.eventId` is non-null, `UPDATE showings SET gcal_event_id = gcal.eventId WHERE id = showing.id`.
   - Wrap GCal block in try/catch — never let GCal failure block the response.
5. Return 201 with the showing record.

**GET handler — list showings for a listing:**

1. `requireProfile()` first.
2. Read `listing_id` from `req.nextUrl.searchParams`. If missing, return 400.
3. Query showings: `.select('id, scheduled_at, status, feedback_json, gcal_event_id, cal_booking_uid, contact:customers(id, name, primary_phone, email), agent:profiles(id, full_name)').eq('listing_id', listingId).eq('org_id', profile.org_id).order('scheduled_at', { ascending: false }).limit(200)`.
4. Return 200 with data array.

<task type="auto">
  <name>Task 2: PATCH + DELETE /api/showings/[id] (SHOW-03 + SHOW-08 update/cancel)</name>
  <files>app/api/showings/[id]/route.ts</files>
  <action>
Create `app/api/showings/[id]/route.ts` with `export const runtime = 'nodejs'`.

**PATCH handler — update status and/or feedback (SHOW-03 + SHOW-08):**

1. `requireProfile()` first. Use `createClient()`.
2. Fetch the existing showing: `.select('id, org_id, gcal_event_id, scheduled_at, status').eq('id', params.id).eq('org_id', profile.org_id).maybeSingle()` — return 404 if not found (own-or-404 pattern, don't leak existence to other orgs).
3. Validate body with Zod:
   ```typescript
   const schema = z.object({
     status:        z.enum(['scheduled','completed','cancelled','no_show']).optional(),
     scheduled_at:  z.string().datetime().optional(),
     feedback_notes: z.string().max(5000).optional(),  // maps to feedback_json.notes
   })
   ```
4. Build the update payload:
   - If `status` provided: include it.
   - If `feedback_notes` provided: set `feedback_json` as merge of existing `feedback_json ?? {}` with `{ notes: body.feedback_notes }` — do NOT overwrite other feedback_json keys.
   - If `scheduled_at` provided: include it (reschedule).
5. Perform the update. Return 200 with updated row.
6. **GCal sync — best-effort, after DB update, in try/catch:**
   - Guard: `if (showing.gcal_event_id) { ... }` — skip if no GCal event was ever created.
   - If status is 'cancelled' or 'no_show': call `updateCalendarEvent(profile.org_id, showing.gcal_event_id, { cancelled: true })`.
   - Else if scheduled_at changed: call `updateCalendarEvent(profile.org_id, showing.gcal_event_id, { startDateTimeIso: body.scheduled_at })`.
   - Never throw from this block — GCal failure is non-fatal.

**DELETE handler — remove showing (SHOW-03 delete path):**

1. `requireProfile()` first.
2. Fetch showing: `.select('id, org_id, gcal_event_id').eq('id', params.id).eq('org_id', profile.org_id).maybeSingle()` — 404 if not found.
3. If `showing.gcal_event_id`: attempt `updateCalendarEvent(profile.org_id, showing.gcal_event_id, { cancelled: true })` in try/catch.
4. Delete: `.delete().eq('id', params.id).eq('org_id', profile.org_id)`.
5. DB trigger `trg_listing_showing_count` fires on DELETE — do NOT manually update showing_count.
6. Return 200 `{ deleted: true }`.
