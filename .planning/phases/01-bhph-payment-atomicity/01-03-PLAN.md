The test strategy:
- Import `POST` from `app/api/pay/[token]/route.ts`.
- Mock `@/lib/supabase/service` to return the `makeTestClient()` supabase instance.
- Mock `@/lib/rateLimit/upstash` so `paymentLimiter` returns `{ allowed: true }`.
- Mock the global `fetch` to simulate Stripe PaymentIntent verify responses.
- Set `supabase.rpc.mockResolvedValueOnce(...)` to control RPC outcomes per test.

The route reads the token from `bhph_payment_tokens` via `supabase.from('bhph_payment_tokens').select(...).eq('token', ...).maybeSingle()`, then reads org settings via `supabase.from('org_settings').select(...).eq(...).maybeSingle()`. Both must be stubbed to return valid data so the route reaches the RPC call.

Use these shared stubs (define once at the top of each describe or in a beforeEach):

```typescript
// Fake payment token record (what the route reads before calling rpc)
const FAKE_TOKEN_ROW = {
  id:               'token-uuid-001',
  amount:           350.00,
  status:           'pending',
  org_id:           ORG_ID,
  customer_id:      'customer-uuid-001',
  bhph_contract_id: 'contract-uuid-001',
}

// Fake org settings (needed to get stripe_dealer_secret_key)
const FAKE_ORG = {
  stripe_dealer_secret_key: 'sk_test_fake',
}

// Fake Stripe PaymentIntent response (returned by the mocked fetch)
const FAKE_PI = {
  id:       'pi_test_001',
  status:   'succeeded',
  amount:   35000,   // 350.00 in cents
  currency: 'usd',
  metadata: { bhph_payment_token: FAKE_TOKEN_ROW.id },
}
```

Helper to build a `NextRequest` for the confirm action:

```typescript
function makeConfirmRequest(payment_intent_id = 'pi_test_001') {
  return new Request('http://localhost/api/pay/test-token', {
    method: 'POST',
    headers: { 'content-type': 'application/json', 'x-forwarded-for': '1.2.3.4' },
    body: JSON.stringify({ action: 'confirm', payment_intent_id }),
  }) as unknown as NextRequest
}
```

**Test cases to write:**

**describe('happy path'):**
- In `beforeEach`: set `supabase.from('bhph_payment_tokens').maybeSingle` to return `{ data: FAKE_TOKEN_ROW, error: null }`, set `supabase.from('org_settings').maybeSingle` to return `{ data: FAKE_ORG, error: null }`, mock global `fetch` to return `{ ok: true, json: () => Promise.resolve(FAKE_PI) }`, set `supabase.rpc.mockResolvedValueOnce({ data: { ok: true }, error: null })`.
- Test: `POST()` returns status 200 with body `{ ok: true }`.
- Test: `supabase.rpc` was called once with `('finalize_bhph_payment', expect.objectContaining({ p_token_id: FAKE_TOKEN_ROW.id, p_stripe_payment_intent: 'pi_test_001' }))`.
- Test: `supabase.from('bhph_payment_tokens').update` was NOT called (verify by checking the stub's `.update` mock call count is 0). This confirms no individual writes.
- Test: `supabase.from('activities').insert` was NOT called.

**describe('idempotency — already_processed'):**
- Set rpc to return `{ data: { already_processed: true }, error: null }`.
- Test: `POST()` returns status 200 with body `{ ok: true, already_processed: true }`.
- Test: no error thrown (the test itself resolves cleanly).

**describe('RPC failure'):**
- Set rpc to return `{ data: null, error: { message: 'DB error', code: '42P01' } }`.
- Test: `POST()` returns status 500 with body `{ error: 'Could not finalize payment' }`.

**describe('conflict — token already processed by different PI'):**
- Set rpc to return `{ data: { conflict: true }, error: null }`.
- Test: `POST()` returns status 409 with body `{ error: 'Token already processed' }`.

**Mocking approach for supabase.from stubs:**

The `makeQueryBuilder()` in testClient returns a stub where every method returns the stub itself. To control what `maybeSingle()` resolves to for a specific call, use:

```typescript
supabase._table('bhph_payment_tokens').maybeSingle
  .mockResolvedValueOnce({ data: FAKE_TOKEN_ROW, error: null })
supabase._table('org_settings').maybeSingle
  .mockResolvedValueOnce({ data: FAKE_ORG, error: null })
```

**Module mock for service client:**

```typescript
vi.mock('@/lib/supabase/service', () => ({
  createServiceClient: () => supabase,
}))
```

Declare `supabase` in the outer scope from `makeTestClient()` so it is accessible in both the mock factory and the test cases.

**Module mock for rate limiter:**

```typescript
vi.mock('@/lib/rateLimit/upstash', () => ({
  paymentLimiter: vi.fn().mockResolvedValue({ allowed: true }),
}))
```

**Global fetch mock:**

```typescript
vi.stubGlobal('fetch', vi.fn())
// In beforeEach for tests that reach Stripe verify:
;(fetch as ReturnType<typeof vi.fn>).mockResolvedValueOnce({
  ok: true,
  json: () => Promise.resolve(FAKE_PI),
})
```

**Important:** The route handler checks `pt.status !== 'pending'` before calling the Stripe verify or RPC. In the idempotency/conflict/failure tests the token row still has `status: 'pending'` — these conditions are returned by the RPC itself, not the pre-check. Do not set the token row to `status: 'paid'` in these tests or the route will 410 before reaching the RPC.
