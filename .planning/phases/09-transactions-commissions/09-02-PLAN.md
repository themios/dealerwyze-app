<task type="auto">
  <name>Task 2: POST/GET /api/transactions and GET/PATCH /api/transactions/[id]</name>
  <files>app/api/transactions/route.ts, app/api/transactions/[id]/route.ts</files>
  <action>
    Create `app/api/transactions/route.ts`:

    **POST** — create a transaction:
    1. `requireProfile()` — reject if no session
    2. Verify `profile.org_id` is from an RE org: check `organizations.vertical = 'real_estate'` via `createClient()`. Return 403 if dealer org.
    3. Parse body with `TransactionCreateSchema`. Return 400 with `{ error: 'Invalid input' }` on failure (no schema details exposed).
    4. Verify `vehicle_id` belongs to this org: query `vehicles WHERE id = vehicle_id AND user_id = profile.org_id`. Return 404 if not found.
    5. Generate `transaction_number`: `TXN-${new Date().getFullYear()}-${String(Math.floor(Math.random() * 9000) + 1000).padStart(4, '0')}` (simple for MVP; collisions acceptable at this scale).
    6. INSERT into transactions with `org_id = profile.org_id`, `pipeline_status = 'offer'`, `status = 'offer'`, and all validated fields. Return 201 with created row.

    **GET** — list transactions for a vehicle:
    1. `requireProfile()`
    2. Read `vehicle_id` from searchParams. Return 400 if missing.
    3. Verify vehicle belongs to org (same check as POST step 4).
    4. Query `transactions WHERE org_id = profile.org_id AND vehicle_id = vehicle_id ORDER BY created_at DESC LIMIT 50`.
    5. Return 200 with `{ transactions: rows }`.

    Create `app/api/transactions/[id]/route.ts`:

    **GET** — fetch single transaction:
    1. `requireProfile()`
    2. Query `transactions WHERE id = params.id AND org_id = profile.org_id`. Return 404 if not found (prefer 404 over 403 — don't confirm existence to wrong tenant).
    3. Return 200 with transaction row.

    **PATCH** — update transaction:
    1. `requireProfile()`
    2. Parse body with `TransactionUpdateSchema`. Return 400 on failure.
    3. If `pipeline_status` is being updated: fetch current row to get current `pipeline_status`. Call `canTransition(current, requested)`. Return 422 `{ error: 'Invalid status transition' }` if false.
    4. Build update object from only the validated fields present in body (no partial overwrites of commission data).
    5. UPDATE `transactions SET ... WHERE id = params.id AND org_id = profile.org_id`. Return 404 if no rows affected.
    6. Return 200 with updated row.

    Across both files:
    - Use `createClient()` (not service client) — RLS handles tenant isolation
    - Never expose Postgres error messages to caller — catch errors and return `{ error: 'Internal error' }` with 500
    - No `console.log` of request body or user data
  </action>

    `curl -X PATCH /api/transactions/[id]` with `pipeline_status: 'closed'` when current is 'offer' returns 422.
    `npx tsc --noEmit` passes.
  </verify>
