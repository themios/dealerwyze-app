<task type="auto">
  <name>Task 3: Migration 195 — close_re_transaction RPC</name>
  <files>supabase/migrations/195_close_re_transaction_rpc.sql</files>
  <action>
    Create migration 195. Implement the `close_re_transaction` Postgres function. This is SECURITY DEFINER so it can update both `transactions` and `vehicles` atomically regardless of caller's RLS context.

    Function signature:
    ```sql
    CREATE OR REPLACE FUNCTION public.close_re_transaction(
      p_org_id UUID,
      p_transaction_id UUID,
      p_closing_price NUMERIC,
      p_closing_date DATE,
      p_closed_by UUID
    )
    RETURNS JSONB
    LANGUAGE plpgsql
    SECURITY DEFINER
    SET search_path = ''
    AS $$
    ```

    Logic inside the function:
    1. SELECT * INTO v_txn FROM public.transactions WHERE id = p_transaction_id AND org_id = p_org_id FOR UPDATE.
       RAISE EXCEPTION 'Transaction not found or access denied' IF NOT FOUND.
       RAISE EXCEPTION 'Transaction is already closed' IF v_txn.pipeline_status = 'closed'.
       RAISE EXCEPTION 'Transaction cannot be closed from current status' IF v_txn.pipeline_status NOT IN ('under_contract','inspection','appraisal','closing').

    2. Fetch commission plan — agent-specific first, then org default:
       ```sql
       SELECT * INTO v_plan
       FROM public.commission_plans
       WHERE org_id = p_org_id
         AND (agent_id = v_txn.listing_agent_id OR (agent_id IS NULL AND is_default = true))
       ORDER BY agent_id NULLS LAST
       LIMIT 1;
       ```

    3. Calculate commission snapshot:
       - gross_commission = ROUND(p_closing_price * COALESCE(v_txn.commission_pct, 3.0) / 100, 2)
       - referral_deduction = COALESCE(v_plan.referral_fee_flat, 0)
       - net_commission_pool = gross_commission - referral_deduction
       - listing_agent_split_pct = COALESCE(v_plan.agent_split_pct, 70)
       - listing_agent_amount = ROUND(net_commission_pool * listing_agent_split_pct / 100, 2)
       - broker_amount = net_commission_pool - listing_agent_amount
       - co_broke_pct = COALESCE(v_plan.co_broke_pct, 0)
       - buyer_agent_amount = ROUND(gross_commission * co_broke_pct / 100, 2)  -- co-broke share of gross, defaults to 0 if no co-broke
       - Build v_snapshot JSONB with: plan_id (v_plan.id or null), plan_type, closing_price, commission_pct, gross_commission, referral_deduction, net_commission_pool, listing_agent_split_pct, listing_agent_amount, broker_amount, co_broke_pct, buyer_agent_amount, calculated_at (now()).

    4. UPDATE public.transactions SET
         pipeline_status = 'closed',
         status = 'closed',
         closing_price = p_closing_price,
         closing_date = p_closing_date,
         commission_snapshot = v_snapshot,
         commission_plan_id = v_plan.id  -- may be null if no plan found
       WHERE id = p_transaction_id AND org_id = p_org_id.

    5. UPDATE public.vehicles SET
         status = 'sold',
         sold_price = p_closing_price,
         sold_at = now()
       WHERE id = v_txn.vehicle_id AND user_id = p_org_id.
       (vehicles uses user_id not org_id for org scoping — this is correct per CLAUDE.md gotchas)

    6. RETURN v_snapshot.

    After the function: GRANT EXECUTE ON FUNCTION public.close_re_transaction TO service_role;

    Use DECLARE section for: v_txn transactions%ROWTYPE; v_plan commission_plans%ROWTYPE; v_snapshot JSONB; gross_commission NUMERIC; referral_deduction NUMERIC; net_commission_pool NUMERIC; listing_agent_split_pct NUMERIC; listing_agent_amount NUMERIC; broker_amount NUMERIC; co_broke_pct NUMERIC; buyer_agent_amount NUMERIC.

    Do NOT use RAISE NOTICE or log any amounts to Postgres logs (no secrets/PII in logs).
  </action>

    Vehicle update verified: after calling the RPC with a test transaction, run `SELECT status FROM vehicles WHERE id = [test_vehicle_id]` and confirm result is 'sold'. This verifies the atomic vehicle status update, not just function existence.
  </verify>
  <done>RPC exists and is callable. SECURITY DEFINER confirmed. Atomically updates both transactions and vehicles in one PL/pgSQL block. Returns JSONB commission_snapshot including buyer_agent_amount. Vehicle status confirmed 'sold' after RPC call.</done>
