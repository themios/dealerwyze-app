<task type="auto">
  <name>Task 1: Commission summary API</name>
  <files>app/api/transactions/summary/route.ts</files>
  <action>
    Create `app/api/transactions/summary/route.ts`:

    **GET** — commission summary:
    1. `requireProfile()` — 401 if no session.
    2. Verify RE org. 403 if dealer.
    3. Read query params: `year` (optional, defaults to current year), `agent_id` (optional, only honored if caller is admin).
    4. Build query:
       - If `isDealerAdmin(profile.role)`: query all closed transactions for org, optionally filtered by agent_id if provided.
         `transactions WHERE org_id = profile.org_id AND pipeline_status = 'closed' AND EXTRACT(YEAR FROM closing_date) = year ORDER BY closing_date DESC LIMIT 500`
       - If agent (not admin): query only transactions where `listing_agent_id = profile.id OR buyer_agent_id = profile.id` (agent sees their own deals only).
         `transactions WHERE org_id = profile.org_id AND (listing_agent_id = profile.id OR buyer_agent_id = profile.id) AND pipeline_status = 'closed' AND EXTRACT(YEAR FROM closing_date) = year ORDER BY closing_date DESC LIMIT 500`
    5. For each transaction: extract from `commission_snapshot` JSONB — `listing_agent_amount`, `buyer_agent_amount` (present when co_broke_pct > 0, defaults to 0 if absent), `gross_commission`, `closing_price`. Join vehicle address via `vehicles.address` (or `vehicles.year || ' ' || vehicles.make || ' ' || vehicles.model` for address field — check schema for RE address column, likely `vehicles.address`).
    6. Compute YTD total:
       - For agents: sum of listing_agent_amount where listing_agent_id = profile.id, plus sum of buyer_agent_amount where buyer_agent_id = profile.id
       - For admins (all-org view): group by agent_id, sum per agent
    7. Return:
       ```json
       {
         "year": 2026,
         "ytd_total": 45000.00,
         "transactions": [
           {
             "id": "uuid",
             "transaction_number": "TXN-2026-0001",
             "closing_date": "2026-05-15",
             "closing_price": 450000,
             "vehicle_address": "123 Main St",
             "gross_commission": 13500,
             "listing_agent_amount": 9450,
             "buyer_agent_amount": 0,
             "broker_amount": 4050,
             "role": "listing_agent"
           }
         ],
         "agents_summary": [  // only if isAdmin — TXN-08 broker view
           { "agent_id": "uuid", "agent_name": "Jane Smith", "ytd_total": 22000, "deal_count": 4 }
         ]
       }
       ```
    8. Cap at 365-day range (year filter covers this). Never unbounded query.
    9. No stack traces in errors. Catch Postgres errors → 500 `{ error: 'Internal error' }`.

    Note: `buyer_agent_amount` is stored in commission_snapshot by the RPC when `co_broke_pct > 0`. For transactions without co-broke, it will be 0. Read it directly from snapshot — do not recalculate.
  </action>
  <verify>
    GET /api/transactions/summary without auth: 401.
    GET as agent: returns only transactions where they are listing or buyer agent. Response includes buyer_agent_amount field (0 if no co-broke).
    GET as admin: returns all org transactions plus agents_summary array.
    `npx tsc --noEmit` passes.
  </verify>
  <done>Summary API returns correct data scoped by role. YTD total computed. Admin gets agents_summary (TXN-08); agents get own deals only (TXN-07). buyer_agent_amount included in all transaction rows.</done>
</task>
