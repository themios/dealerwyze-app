<task type="auto">
  <name>Task 1: Close transaction API route and CloseTransactionDialog</name>
  <files>app/api/transactions/[id]/close/route.ts, components/transactions/CloseTransactionDialog.tsx</files>
  <action>
    Create `app/api/transactions/[id]/close/route.ts`:

    **POST** — close a transaction:
    1. `requireProfile()` — reject 401 if no session.
    2. Verify RE org (vertical = 'real_estate'). 403 if dealer.
    3. `isDealerAdmin(profile.role)` — return 403 `{ error: 'Only brokers and admins can close transactions' }` if false. (Agents cannot close.)
    4. Parse body with Zod schema: `{ closing_price: z.number().positive(), closing_date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/) }`. Return 400 on failure.
    5. Verify transaction belongs to org: query `transactions WHERE id = params.id AND org_id = profile.org_id`. Return 404 if not found. Return 422 if pipeline_status is already 'closed'.
    6. Call `supabase.rpc('close_re_transaction', { p_org_id: profile.org_id, p_transaction_id: params.id, p_closing_price: closing_price, p_closing_date: closing_date, p_closed_by: profile.id })`.
       - Use `createClient()` (not service client) — the RPC is SECURITY DEFINER so it will bypass RLS internally, but the route itself should operate in the authenticated user context.
    7. On RPC error: check error message. Map known messages to clean user-facing errors (e.g., 'Already closed' → 422, 'Transaction not found' → 404, others → 500 `{ error: 'Failed to close transaction' }`). Never forward Postgres error details.
    8. On success: return 200 `{ transaction_id: params.id, commission_snapshot: rpc_result }`.

    Create `components/transactions/CloseTransactionDialog.tsx` — client component:
    - Props: `transaction: Transaction`, `onClosed: (snapshot: Record<string, unknown>) => void`, `open: boolean`, `onOpenChange: (open: boolean) => void`
    - Dialog content: "Close Transaction" heading, warning text ("This action is permanent and cannot be undone"), closing price input (number), closing date input (date), confirm button.
    - On submit: POST to `/api/transactions/${transaction.id}/close`. Show loading state.
    - On success: call `onClosed(snapshot)` with returned commission_snapshot. Close dialog.
    - On error: show error message inline. Never blank dialog on error.
    - After successful close, display commission breakdown from snapshot:
      - Gross commission: $X
      - Referral deduction: $X (if > 0)
      - Net commission pool: $X
      - Agent amount (listing): $X (split_pct%)
      - Broker amount: $X
    - Use shadcn/ui Dialog component (already in codebase).

    Wire into TransactionPanel (in `components/transactions/TransactionPanel.tsx`):
    - Add "Close Transaction" button visible only when `isAdmin = true` and `currentStage` is not 'closed' or 'fallen_through'.
    - Renders `<CloseTransactionDialog>` when clicked.
    - After close callback: refresh transactions list.
  </action>
  <verify>
    POST /api/transactions/[id]/close with agent auth: 403.
    POST with admin auth, closed transaction: 422.
    POST with admin auth, valid closing_price and closing_date: 200 with commission_snapshot.
    `npx tsc --noEmit` passes.
  </verify>
  <done>Close API calls RPC correctly. Broker gate enforced. Commission snapshot returned. CloseTransactionDialog handles success and error states. TransactionPanel shows Close button for admins.</done>
